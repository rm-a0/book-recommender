Strategies
Stage 1

Ingestion + Cleaning

Ratings
1.1M to 433k +
  • drop nulls - User-ID, ISBN, Rating
  • drop duplicate (User-ID, ISBN) pairs
  • drop 715k implicit 0-ratings - disliked vs unread indistinguishable
  • min 3 ratings per user
Books
271k to 22k +
  • drop nulls - ISBN, Book-Title
  • lowercase + strip - titles and authors
  • clean year - raw has "DK Publishing", 0, 2037
  • drop books with no ratings after ratings clean
  • min 5 raters per book - CF stat reliability
  • deduplicate editions - same title+author, keep lowest ISBN
Users
278k kept +
  • drop null User-ID
  • clamp age 13-100 - 110k nulls, values like 0 and 200+
  • string to double to int - handles "25.0" format
Improvements
  • editions keep only one ISBN - aggregate ratings across all editions instead
  • fixed min-5 threshold - dynamic filter based on rater activity sum
  • age as demographic proxy - cluster users by rating behaviour
  • no evaluation metric - tune thresholds against precision@k or NDCG
Stage 2

Collaborative Filtering

O(1) lookup table
  • isbn_index.json ISBN -> integer matrix position
  • item_similarity.npz scipy CSR sparse matrix
  • inference: one row slice, no DB query, no recomputation
  • reused by ReadersAlso, HiddenGems, TopPicks
Build (offline, PySpark)
  • self-join on User-ID - all co-rated book pairs
  • filter isbn_a < isbn_b - each pair once
  • adjusted cosine - subtract user mean before dot product
  • removes per-user scale bias
  • store only positive similarities
Improvements
  • no confidence weighting - sim over 2 shared raters equals sim over 200, fix: shrinkage sim x n/(n+lambda)
  • item-item only - add user-user matrix, reweight at inference by rater similarity to query user
  • explicit CF - replace with NCF or ALS once feedback loop exists
  • two-tower - user + item towers in shared embedding space (see prod arch)
Stage 3

Open Library Enrichment

Why
  • raw dataset - title, author, year, publisher only
  • semantic search needs descriptions + subject tags
  • without enrichment - every book embeds as "Title by Author"
How
  • batch - 100 ISBNs per request, 220 requests not 22,000
  • concurrent - 10 batches in parallel, 100/min rate limit
  • cache - openlibrary_cache.json, reruns skip fetched
  • flush every 1000 ISBNs - crash recovery
  • fallback - "Title by Author" if no entry found
Improvements
  • partial coverage - switch to Google Books API or Open Library bulk download
  • static scraping - Goodreads / Amazon for richer descriptions
  • title-only fallback is weak - LLM-generated descriptions for uncovered books
  • subjects comma-separated string - store as array, enable tag-based filtering
Output: book_metadata_enriched.parquet
  • ISBN, title, author
  • description - Open Library or fallback
  • subjects - up to 20 tags, comma-separated
Stage 4

Semantic Embeddings

Input
  • "Title by Author. Description. Genres: subjects"
  • title+author always present as anchor
  • description skipped if equals fallback
  • "Genres:" prefix - signals categorical not prose
Model + Index
  • all-MiniLM-L6-v2 - 384-dim, fast, strong zero-shot
  • L2 normalize - inner product becomes cosine similarity
  • FAISS IndexFlatIP - exact search, no approximation
  • 22k books - brute force under 1ms per query
  • ISBN map - position -> ISBN, FAISS returns positions not strings
Improvements
  • in-memory index - every pod loads its own copy, move to MongoDB Atlas or Pinecone
  • exact search at scale - switch IndexFlatIP to IndexIVFPQ at 1M+ books
  • MiniLM quality ceiling - E5 or MPNet for better embeddings
  • text-only embeddings - add cover image features (multi-modal)
  • generic pretrain - fine-tune on book corpus or interaction data

Results

Evaluation

Works

  • CF strong for popular books - dense rating overlap
  • semantic - cross-genre thematic matches, zero shared raters needed
  • Hidden Gems - genuinely underrated books surface
  • SameAuthor - deterministic, no ML failure modes
  • six strategies complement each other
  • TopPicks quality gate filters low-quality noise
  • edition filter (rapidfuzz) removes same-book duplicates

Breaks

  • niche books - CF collapses, dataset sparsity not algorithm
  • no descriptions - title-only embedding weak signal
  • age group - coarse proxy, age does not equal taste
  • edition rating split - same book, different ISBNs
  • no eval metric - thresholds pragmatic not empirically tuned
  • no user identity - seed-only, no personalisation
  • cold start for new books - CF fails until enough ratings
